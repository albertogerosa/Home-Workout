# Home-Workout

## Cognome Nome


Titolo
home workout
Tagline
Palestra virtuale senza l'utilizzo di attrezzi costosi.
Descrizione
 un app per aiutare le persone a mantenersi in forma da casa, per chi ha poco tempo, senza bisogno di andare in palestra o usare attrezzature costose. L’app propone allenamenti brevi, e accessibili a tutti i livelli
Target
persone con poco tempo a disposizione per andare in palestra e persone che non hanno la disponibilità economica di comprarsi dell'attrezzatuira da allenamento o un abbonamento in palestra
Competitors
altre applicazioni sull'allenamento fisico
Tecnologie
html, js, css, php, sql
Link pubblico alla web app
requisiti
RF1 – Registrazione e Login
RF1.1: L’utente deve potersi registrare tramite email e password.
RF1.2: L’utente deve poter effettuare il login.
RF1.3: L’app deve gestire il recupero password.
RF2 – Profilo Utente
RF2.1: L’utente deve poter inserire dati personali (nome, età, livello fitness, obiettivi).
RF2.2: L’utente deve poter modificare i propri dati.
RF2.3: L’app deve salvare lo storico degli allenamenti.
RF3 – Allenamenti
RF3.1: L’app deve proporre allenamenti senza attrezzatura.
RF3.2: Gli allenamenti devono essere suddivisi per livello (principiante, intermedio, avanzato).
RF3.3: Devono essere presenti sessioni brevi (5–20 minuti).
RF3.4: L’utente deve poter visualizzare esercizi con:
descrizione testuale
immagini o animazioni
timer o numero di ripetizioni
RF3.5: L'app deve generare allenamenti personalizzati sulla base del profilo utente.
RF4 – Programmi di Allenamento
RF4.1: L’app deve offrire programmi settimanali o mensili preimpostati.
RF4.2: L’utente deve poter avviare e completare un programma.
RF5 – Timer e Funzionalità Interattive
RF5.1: L’app deve includere un timer integrato per esercizi e circuiti.
RF5.2: Durante l’allenamento l’utente deve poter mettere in pausa o saltare un esercizio.
RF6 – Dashboard
RF6.1: L’utente deve poter visualizzare i propri progressi (allenamenti conclusi, tempo allenamento, calorie stimate).
RF6.2: La dashboard deve mostrare gli allenamenti consigliati del giorno.
RF7 – Sistema di Feedback
RF7.1: L’utente deve poter valutare gli allenamenti (es. difficoltà percepita).
RF7.2: L’app deve adattarsi in base al feedback.
RF8 – Notifiche / Reminders
RF8.1: L’utente deve poter ricevere promemoria per allenarsi.
RF8.2: Le notifiche devono essere configurabili.
